SE Hub — 5 dashboard concepts

Five different ways to organize the platform: customer repository (transcripts → facts → architecture) + Current State / Future State diagrams. Same demo data across all 5 so you're choosing the layout, not the content. Pick one (or "1 + 4 mash-up") and we'll build it.

1CRM Workspace
Salesforce-Lightning style
Left: customer list. Center: tabbed detail (Overview / Current / Future / Transcripts / Recs). Right: live activity. Familiar; SEs feel at home day one.
+ Familiar muscle memory · scales to 100s of accounts
— A bit "spreadsheet of customers"; less visual punch on day one
2Pipeline Kanban
Sales-stage cards
Customer cards grouped by deal stage (Discovery → POC → Negotiation → Closed). Each card has a mini architecture thumbnail + key stats. Click → opens detail.
+ Sales-pipeline-native · spatial, scannable · "where's the deal?"
— Less optimized for the "I have one customer right now" workflow
3Big-Picture Console
Aggregate-first
Hero: portfolio stats (customers, VMs protected, replaced competitors). Below: searchable grid of customer tiles, each with mini-diagram thumbnails. Drill down to detail.
+ Manager-friendly · pretty for QBRs · visual
— More clicks to get to per-customer work
4Split-Diagram Workspace
Architecture-first
Customer selector at top. Main view: 50/50 split with Current State on left, Future State on right. Bottom drawer: transcript upload + extracted facts. Single-customer focus.
+ The "do real work" surface · diagram-first · transcripts feed directly into the right panel
— Doesn't help a manager see the whole portfolio at a glance
5AI-First Conversational
Copilot-style
Chat interface center stage ("show me Acme", "draft a future state", "compare to industry"). Sidebar = customer context. Diagrams appear inline in the conversation.
+ Natural-language native · scales to "ask anything" · diagrams as artifacts
— Higher trust barrier · takes a few weeks to feel productive
